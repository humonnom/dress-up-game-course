## 📺 완성본 시연 (2분)

자, 먼저 우리가 만들 게임이 어떤 모습인지 보여드릴게요.

*[화면: 완성된 게임 실행]*

보시는 것처럼, 왼쪽에는 캐릭터가 있고, 오른쪽에는 다양한 옷과 액세서리 아이템들이 있습니다.

*[아이템을 드래그해서 캐릭터에게 입혀보기]*

- 아이템 보드에서 아이템을 드래그해서 캐릭터에게 드롭하면 옷을 입힐 수 있어요.
- 같은 카테고리의 아이템은 하나만 입을 수 있습니다. 예를 들어 상의를 새로 입히면 기존 상의는 자동으로 착용 해제됩니다.
- 이미 착용한 아이템은 마우스로 드래그해서 위치를 조정할 수 있어요.
- 캐릭터 영역 바깥으로 아이템을 드래그하면 착용 해제됩니다.

*[다양한 조합 시연]*

머리, 상의, 아우터, 바지, 양말, 신발, 가방, 액세서리까지 다양하게 조합할 수 있죠.

## 🗂️ 프로젝트 구조 설명 (1분 30초)

이제 프로젝트 구조를 살펴볼게요.

*[화면: 파일 탐색기]*

우리 프로젝트는 이렇게 구성됩니다:

```
dress-up-game-js/
├── index.html          # 메인 HTML 파일
├── css/
│   └── style.css       # 스타일 시트
├── js/
│   └── app.js          # 메인 JavaScript 파일
└── img/
    └── svgs/           # SVG 이미지 파일들
        ├── character.svg
        ├── hair/
        ├── clothes/
        ├── shoes/
        └── item/
```

- **index.html**: 게임의 기본 구조를 담당합니다
- **css/style.css**: 모든 스타일링을 여기서 관리합니다
- **js/app.js**: 게임의 모든 로직이 들어갑니다
- **img/svgs/**: SVG 형식의 캐릭터와 아이템 이미지들이 있습니다

## 🎨 SVG 이미지 파일 설명 (1분)

이 프로젝트의 특별한 점은 SVG 이미지를 사용한다는 거예요.

각 아이템은 두 가지 버전이 있습니다:

1. **아이템 보드용** (`jeans.svg`): 아이템 보드에 표시되는 이미지
2. **착용용** (`jeans-on-body.svg`): 캐릭터가 실제로 입었을 때의 이미지

*[화면: 파일 탐색기에서 두 파일 비교]*

왜 두 개가 필요할까요? 아이템 보드에서는 아이템 자체만 보여주지만, 캐릭터에게 입혔을 때는 캐릭터의 몸에 맞춰진 모양이 필요하기 때문입니다.

특별한 아이템도 있어요. 백팩 같은 경우는:
- `backpack-on-body-back.svg`: 캐릭터 뒤쪽
- `backpack-on-body-front.svg`: 캐릭터 앞쪽

이렇게 두 개의 레이어로 나뉘어서 입체감을 표현합니다.
